## FAIL — 제품 결함 (환경 무관)

### P1
- **#1 결제 진입 불가** — `settings/page.tsx:170` Upgrade가 `<a href="/api/checkout">`(GET)인데 라우트는 POST 전용 → **라이브 405 확인**. 수익화 경로 전면 차단.
- **#2 리포트 내보내기 불가** — `report/[id]/page.tsx:70`이 `window.open('/api/export?...')`(GET), 라우트는 POST 전용 → **라이브 405 확인**.
- **#3 영수증 상세가 죽은 엔드포인트 호출** — `receipt/[id]/page.tsx:319`가 `/api/receipt/{id}` 호출하나 해당 라우트 부재(실제 라우트는 `/api/calculate/[id]`) → 라이브 400.
- **#4 motor-efficiency 정격부하에서 η² 반환** — 부하율 1.0에서 84.8% = 0.921²(표값 92.1%). 효율이 부하 증가에 따라 **감소**하는 물리 역전(0.25→90.2%, 1.0→84.8%). 게다가 신설 accuracy 테스트가 틀린 86.5%를 known answer로 고정(닫힌 순환).
- **#5 ampacity-compare 날조 잔존(사용자 경로)** — `/calc` 도달 `ampacity-compare.ts:59-61`의 `NEC×0.98 / IEC×1.02` 그대로. 라이브: nec=135·iec=141인데 리포 자체 NEC 310.16 표는 4AWG=95A·3AWG=115A. 앞선 수리는 호출자 0인 `standard-comparator`에만 적용됨(wired 착시).

### P2
- **#6 impedance-voltage 출고 기본값** — `calculator-params.ts` 기본 Isc=100A로 실행 시 **%Z = 759.67%**(물리 불가). 공식은 고쳤으나 기본 입력 미수정.
- **#7 nearestNecSize 상향 스냅** — 25mm²를 3AWG(26.67mm², 115A)로 매핑. 열적 한계에서 비보수 방향(+21%). 4AWG(95A) 하향이 맞음.

### P3
- **#8 중복 렌더 의심** — 계산기 페이지 DOM에 "계산하기"·"입력값 포함 링크 복사" 버튼이 각 2벌(반응형 변형으로 추정, 접근성·테스트 혼선 가능).
- **#9 Next 16 deprecation** — `middleware` 규약 사용 중, `proxy`로 이전 권고(부팅 경고).
