   ▼                          ▼                    ▼

   │                          │                    │
   ▼                          ▼                    ▼
[파서]  Vision LLM        parseDxfToSLD       parsePdfToSLD
        (외부 모델)       dxf-parser.ts       pdf-vector-parser.ts

   ▼

## 2. 성능 (라이브 실측)

| 항목 | 실측 | 상태 |
|---|---|---|
| **판독 — 실발주 도면** | KIMM 분전반: `MCCB 3P-250/150`·`CT (A)`·`CAPACITOR(uF)` 완성형 정확 추출. p17 심볼 개수 근사 일치(차단기 2=2·계기 1=1) | ✅ |
| **판독 — 교재/산문 도면** | dsan: "TR"·"DS"가 설명 **문장**에서 오분류(label이 산문). 단 conf 0.55 강등+"배치만 참고"로 신호됨 | ⚠️ |
| **confidence 4단계** | 0/0.3/0.55/0.85 + anchored(snapped>junctioned) 강등. 실도면 양방향 전환(KIMM 0.85 ↔ dsan/PMG 0.55) = 열린 순환 | ✅ |
| **정직 거부** | 래스터 2종(KBRI 16.5MB·CODIL 466p) → 400. 비PDF → 400. 크래시 0 | ✅ |
| **처리 속도** | KIMM 24MB p2(선분 800·컴포넌트 89) ~225ms. 대용량 프록시 절단 없음(100mb) | ✅ |
| **판독 정확도(정량)** | 골든 라벨 없어 재현율/정밀도 **미측정**. 렌더 도구(pdftoppm) 부재로 픽셀 대조 불가 | ⏸ |
| **DoS 내성** | 파싱 타임아웃·연산자/선분/앵커 개수 캡 부재 + O(N²) 스냅. 1.4MB 합성 PDF로 ~10초 이벤트루프 점유(독립 심사 재현) | ⏸ 릴리스 이월 |

기계 게이트: `npm run gate:pdf` **9/9** · `topology` 테스트 **75/75** · 전체 jest **749/749** · build 0.

## 5. 약점·이월 (정직 선언)

1. **교재/산문 오분류** ⚠️ — `TR`·`DS`가 설명 문장에서 단어로 잡혀 컴포넌트화. label이 문장 수준으로 길다는 특성으로 필터 가능(임계 발명 없이 "심볼=단문"). conf 0.55 강등이 1차 방어. **다음 수리 후보**.
2. **판독 정확도 미인증** ⏸ — 파싱이 도는 것(수용성)은 실증. "얼마나 맞나"는 사람 골든 라벨 필요. 다음 단계: KIMM 1~2장 사람 대조.
3. **DoS** ⏸ — 파서 O(N²)+무타임아웃. 미배포 내부도구라 릴리스 하드닝(worker 타임아웃·개수 캡)으로 이월. GPT가 rate-limit 메모리 무한증가는 부분 차단.
4. **곡선 결선 미추출** ⚠️(NIT) — curveTo는 끝점만 이동, 곡선 도체 결선 누락 가능. 실 SLD 배선은 직선이라 실측상 미발생(문서화된 설계).
5. **회전 페이지** ⚠️(NIT) — `/Rotate` 미반영. 회전 픽스처 부재로 미측정.
