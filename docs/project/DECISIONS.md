# ESA 구조 결정 기록

## 2026-07-20 — 판정은 근거가 없으면 HOLD

- 기준서 제목이나 검색 결과가 있다는 사실만으로 PASS를 만들지 않는다.
- 전용 평가기, 입력 임계값, 공인 판본 근거 중 필수 요소가 없으면 HOLD와 누락 근거를 반환한다.
- 저장된 판본 문자열만으로 `isStandardCurrent=true`를 만들지 않는다.

## 2026-07-20 — 합의는 전문팀 결과와 분리

- 계통도팀, 평면도팀, 기준서팀은 서로 다른 입력 관점과 산출물을 가진다.
- 합의 단계는 네 번째 독립 전문가가 아니라 세 결과의 충돌을 정리하는 별도 단계다.
- 서로 다른 전문팀 두 곳 이상이 정상 완료되지 않으면 합의 완료 상태를 만들지 않는다.

## 2026-07-20 — 도면 물리 근거와 표시 근거를 동일하게 유지

- 이미지 도면은 실제 픽셀 크롭과 영역 좌표 복원을 사용한다.
- DXF는 `$INSUNITS` 또는 명시 단위가 있을 때만 물리 길이를 계산한다.
- 보고서 화면은 실제 검출 좌표가 없는 임의 마킹을 그리지 않고 근거 목록을 보여 준다.
- 명판값·부하 조건이 없으면 합성 전류·길이로 실제 계산기를 실행하지 않는다.

## 2026-07-20 — BYOK는 브라우저 암호화와 서버 일시 전달

- 평문 키는 localStorage, DB, 로그, URL, 응답에 기록하지 않는다.
- 암호화 키는 IndexedDB의 추출 불가능 `CryptoKey`, 암호문은 localStorage에 둔다.
- 공급자 호출 시 원문 키는 TLS로 ESA 서버를 통과하므로 “서버를 전혀 통과하지 않는다”고 표현하지 않는다.
- 연결 확인은 same-origin API가 공급자별 고정 공식 엔드포인트만 호출한다.

## 2026-07-20 — 운영 저장은 fail-closed

- 개발 환경의 메모리 폴백은 허용하되 운영에서 DB 누락·실패를 저장 성공으로 반환하지 않는다.
- 영수증과 보고서는 write 뒤 별도 조회에서 해시와 소유권을 다시 확인한다.
- 알림, 감사로그, 구독 티어도 서버 검증 UID와 지속 저장소를 정본으로 사용한다.

## 2026-07-20 — 결제 권한은 서명 웹훅과 DB가 정본

- 브라우저는 Stripe price ID를 보내지 않고 서버 플랜 핸들만 선택한다.
- 서명 검증 웹훅을 멱등 처리한 뒤에만 사용자 티어를 갱신한다.
- 키, 가격, 웹훅 secret, DB가 모두 준비되지 않으면 결제 UI와 API를 비활성화한다.

## 2026-07-20 — IPFS 등록은 공증이 아님

- 기존 `/api/notarize`와 `RECEIPT_NOTARIZE` 이름은 호환 목적으로만 유지한다.
- 사용자 문구와 문서에는 “IPFS 타임스탬프 등록”으로 표시하며 블록체인 거래, 제3자 공증, 법적 서명으로 주장하지 않는다.
- 입력은 깊이 제한과 민감 키·자격증명·연락처 제거를 거친 최소 데이터만 고정한다.

## 2026-07-20 — 미배선 표면은 삭제하거나 휴면 대장에 기록

- 구형 AI provider 서비스, 중복 스트리밍 구현, YouTube 501 API, 가짜 도면 오버레이, 성공할 수 없는 로컬 Vision 선택지를 삭제했다.
- 제품 결정을 기다리는 실구현 API와 엔진만 `docs/DORMANT_MANIFEST.md`에 상태·사유·활성 조건을 기록해 남긴다.

## 2026-07-20 — SLD 판독은 역할별 원본 격리 뒤 메인 합산

- 하나의 모델 응답을 여러 전문가 의견처럼 복제하지 않는다. `symbols`, `connections`, `text`, `logic`은 서로 다른 프롬프트와 소스 계획으로 호출한다.
- 각 역할은 drawing hash, source ID, provider, model, prompt version, output hash가 봉인된 봉투를 반환한다.
- 메인 합산은 역할 출력을 그대로 믿지 않고 현재 도면 그래프에 유일하게 대응되는 근거만 정규화한다.
- 역할 누락, 모호한 ID·소유자·페이지, 독립 논리와 그래프의 충돌은 추천 문장으로 덮지 않고 HOLD 또는 FAIL로 남긴다.

## 2026-07-20 — 원본 표시와 95% 주장은 서로 다른 신뢰 경계

- 원본 도면 바이트는 브라우저 IndexedDB에만 저장하고 SHA-256이 보고서 source hash와 일치할 때만 object URL로 다시 연다.
- 보고서의 `Sxx`·`Lxx` 번호는 기호명과 별개인 추적 ID다. 사용자는 표와 원본 오버레이에서 같은 번호로 관계를 확인한다.
- `verified95`는 UI 점수나 합성 fixture 통과로 만들지 않는다. 승인된 정확한 데이터셋 집합, 실도면 독립 라벨, 평가기 버전, 예측·라벨 해시, Ed25519 서명, 최신 receipt가 모두 필요하다.
- 실증 입력이 없으면 구현 품질과 별개로 95% 주장은 HOLD다.

## 2026-07-21 — V3 전체 판독 완료는 역할·공간·운영 영수증의 교집합

- 벡터 결과 배열이 존재한다는 사실만으로 5개 역할 완료를 만들지 않는다. 파서·토폴로지 검증 뒤 역할별 `vectorAudit` 영수증이 있어야 coverage를 완료한다.
- PDF는 전체 파일을 선렌더하지 않고 요청 페이지를 페이지·총 픽셀·시간·취소 예산 안에서 순차 준비한다.
- “도면에 없음” 제안은 전체 관련 coverage가 끝난 경우에만 SUPPORTED가 될 수 있다. 부분 판독에서는 HOLD다.
- 페이지 간 연결은 확정 OCR, 엄격한 기기 종류, 양쪽 전압 근거, 방향을 함께 요구한다. 같은 라벨이나 누락 전압만으로 자동 병합하지 않는다.

## 2026-07-21 — 취소·재개와 사용자 정정은 내구성과 경쟁 제어를 요구

- 운영 취소·재개는 `DRAWING_JOB_STORE_DIR` 공유 내구 볼륨과 안정적인 `DRAWING_SOURCE_LEASE_SECRET`이 모두 있을 때만 연다.
- 원본 임대는 owner 결박 AES-256-GCM 암호문으로만 저장하고 COMPLETE·CANCELLED 뒤 삭제한다.
- 사용자 정정은 문자, 기기 종류, 기기명을 분리하며 문서 버전 compare-and-swap과 idempotency key를 요구한다.
- 브라우저는 작업 ID만 sessionStorage에 보존하고 GET 폴링으로 결과를 복구한다. 원본 바이트는 보고서 JSON에 넣지 않는다.

## 2026-07-21 — V3 평가 영수증은 구형 manifest 영수증과 호환으로 간주하지 않는다

- V3는 `EvalResult` 계약의 지표·strata·생산 지문·공급자·모델·실제 데이터셋별 3회 실행·Ed25519 서명을 사용한다.
- 서명 범위 밖의 `passesAllThresholds`와 `failedMetrics`는 배지 판정 입력으로 신뢰하지 않고 서명된 지표에서 다시 계산한다.
- `scripts/sld-golden-gate.mjs`의 구형 보고서 영수증은 V3 배지 입력으로 사용할 수 없다.
