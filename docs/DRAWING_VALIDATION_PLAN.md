# 도면 파서 검증 계획 — 교보재 15장 (초5·중5·고5)

작성 2026-07-20 · 대상 `src/engine/topology/{dxf-parser,pdf-vector-parser,endpoint-snap}.ts`

## 왜 지금인가

주력 기능인데 **테스트가 0개다**. `grep -rln "parseDxf" --include=*.test.ts` → 결과 없음. 리포 안에 도면 파일도 0장. 직전에 잡은 잠복 버그 3건(파서 생성자 미생성 · LINE 좌표 필드 불일치로 결선 추출 0 고정 · 끝점 미접합)은 전부 손으로 돌려보다 발견한 것이지 테스트가 잡은 게 아니다. 같은 계열이 더 있는지 현재 알 방법이 없다.

## 난이도 축을 둘로 나눈다

교보재는 **전기 복잡도**로 초·중·고가 매겨져 있다. 그런데 파서를 실제로 깨뜨리는 건 전기 복잡도가 아니라 **파일 난이도**(CAD 방언·레이어 난립·블록 중첩·회전 텍스트·스캔 여부)다. 직전 3버그 중 2건이 파일 난이도 계열이었다.

따라서 교보재의 초·중·고를 티어로 쓰되, **각 티어 안에서 파일 난이도를 의도적으로 섞는다.** 초급 5장을 전부 깨끗한 DXF로 채우면 티어를 통과해도 아무것도 증명하지 못한다.

| 티어 | 전기 복잡도 | 규모 기준 | 파일 난이도 배분(5장) |
|---|---|---|---|
| 초 | 단상 2선식·소규모 분전반 | 노드 ≤ 15 | 깨끗한 DXF 3 · 벡터 PDF 2 |
| 중 | 3상 4선식·변압기 1뱅크·MCC | 노드 15~50 | DXF 2 · 구버전(R12) DXF 1 · 벡터 PDF 1 · 블록 중첩 DXF 1 |
| 고 | 수전설비·다중 뱅크·비상발전·UPS | 노드 50+ | DXF 2 · 회전/척도 비표준 1 · 벡터 PDF 1 · 스캔 래스터 1 |

스캔 래스터 1장은 DXF/PDF 파서가 아니라 **VLM 경로**(`sld-recognition`)를 탄다. 같은 정답 라벨로 두 경로를 비교하는 것이 목적이며, 여기서 갈리면 경로별 강약이 처음으로 드러난다.

## 조달 — 3계층, 계층마다 취급이 다르다

**1층 합성(내가 그림)** — LibreCAD/QCAD 무료 툴로 KEC 관례대로 작도 → DXF 출력. **정답을 우리가 안다**는 게 결정적이다. 노드 12·결선 14로 그렸으면 파서가 12/14를 못 뽑는 순간 버그다. 라이선스 100% 우리 것이라 리포 커밋 자유. → `fixtures/drawings/synthetic/`

**2층 공공·오픈소스** — 한전 표준 결선도, 조달청 공개 입찰 첨부 도면, 폴리텍·마이스터고 실습 교재, 한국전기기술인협회 KEC 해설 예제, 한국산업인력공단 전기기사 실기 기출 도면, 해외는 미 연방 저작물(퍼블릭 도메인)·IEEE/DOE 교육자료. 출처·라이선스를 라벨에 명시하면 커밋 가능. → `fixtures/drawings/public/`

**3층 인터넷 교보재** — 받아서 돌려보고 **결과 수치만 기록, 파일은 커밋 금지**. 폴더를 `.gitignore`에 미리 박아 실수로 올라가는 사고를 구조적으로 차단한다. → `fixtures/drawings/local/` (gitignored)

취급을 나누는 이유는 저작권만이 아니다. 실제 프로젝트 도면에는 발주처명·현장 주소·설계사 인장이 박혀 있어 영업비밀·개인정보 문제가 따라붙는다. **로컬에서 읽고 버리는 것과 리포에 박아 재배포하는 것은 완전히 다른 행위다.**

## 정답 라벨 — 티어마다 방법이 다르다

노드 200개짜리 도면을 손으로 전수 라벨링하면 사람이 틀린다. 그래서:

- **초·중: 전수 라벨** — 모든 노드(id·type·label)와 모든 결선(from·to)을 기록.
- **고: 표본 + 불변식** — 핵심 간선(수전→변압기→주차단기→각 뱅크)만 전수 라벨하고, 나머지는 그래프 불변식으로 검사한다. 고아 노드 0 · 전원에서 모든 부하로 도달 가능 · 결선이 존재하지 않는 노드를 참조하지 않음 · 자기 자신 연결 0.

불변식 검사는 전수 라벨보다 약하지만 **사람이 틀리지 않는다**는 강점이 있다. 직전 "끝점이 허공에 뜬" 버그는 정확히 이 불변식(고아 노드 0)으로 잡혔을 버그다.

라벨 스키마 — 도면 파일 옆에 같은 이름 `.label.json`:

```json
{
  "source": "한국산업인력공단 전기기사 실기 2019-2회",
  "license": "public-institution | PD | local-only",
  "tier": "중",
  "fileDifficulty": ["dxf-r12", "block-nested"],
  "labelMode": "full | sampled",
  "expected": {
    "nodeCount": 24,
    "edgeCount": 27,
    "nodes": [{ "type": "transformer", "label": "TR-1", "rating": "500kVA" }],
    "edges": [{ "from": "TR-1", "to": "ACB-1" }]
  },
  "invariants": { "orphanNodes": 0, "danglingEdges": 0, "selfLoops": 0 }
}
```

`type`은 기존 `SLDComponentType` 14종(transformer·breaker·cable·bus·generator·motor·capacitor·load·switch·relay·meter·panel·ups·mcc)만 쓴다. 새 어휘를 만들지 않는다.

## 지표 — "파싱 성공"은 지표가 아니다

파일을 읽고 예외 없이 끝나는 것과 **맞게 읽는 것**은 다르다. 측정 항목:

| 지표 | 정의 | 무엇을 잡나 |
|---|---|---|
| 노드 재현율 | 잡은 정답 노드 / 실제 노드 | 누락 |
| 노드 정밀도 | 잡은 정답 노드 / 파서가 낸 노드 | 헛것 검출 |
| 결선 재현율·정밀도 | 위와 동일 | 배선 오독 |
| 타입 정확도 | 타입까지 맞은 노드 / 위치가 맞은 노드 | 심볼 오분류 |
| 고아 노드율 | 결선 0인 노드 / 전체 | 끝점 미접합 회귀 |

**1차 실행의 목적은 합격이 아니라 기준선 측정이다.** 지금 임계값을 정하면 근거 없는 숫자를 발명하는 것이 된다(추정 금지 원칙). 15장 측정이 끝난 뒤 실측 분포를 보고 티어별 합격선을 고정하고, 그때부터 회귀 차단용으로 잠근다.

## 단계

**Phase 0 — 하네스 (도면 0장으로 지금 가능)**
라벨 스키마 타입 · 지표 계산기(`compareToLabel`) · 픽스처 러너 · `fixtures/drawings/` 3폴더와 `.gitignore` 배선. 도면이 한 장도 없어도 전부 만들 수 있고, 이게 있어야 이후 전부가 자동화된다.

**Phase 1 — 초급 5장**
합성 3 + 공공 2. 첫 기준선 측정. 여기서 이미 깨지면 중·고로 갈 필요가 없다.

**Phase 2 — 중급 5장** — CAD 방언(R12·블록 중첩)이 처음 들어가는 지점. 파일 난이도 버그가 나온다면 여기다.

**Phase 3 — 고급 5장** — 불변식 검사 본격 가동. 스캔 1장으로 VLM 경로 대조.

**Phase 4 — 임계 고정 + CI 배선**
측정 분포 기반으로 티어별 합격선 확정 → `npm run test:drawings` → CI exit code 게이트. 여기까지 와야 회귀가 기계로 막힌다(🟥).

## 예상 실패 모드 (미리 적어두고 맞는지 본다)

미리 적어두는 이유는 나중에 결과를 보고 "그럴 줄 알았다"고 사후 합리화하는 것을 막기 위함이다.

1. 블록(INSERT) 중첩 — 심볼이 블록 안에 있으면 노드로 안 잡힘 (중급에서 발현 예상)
2. 척도·회전 — 좌표 변환 누락 시 끝점 접합 허용오차(bbox 대각선 5%)를 벗어나 고아 대량 발생 (고급)
3. 텍스트-심볼 연결 — 정격(500kVA)이 별도 TEXT 엔티티라 노드에 안 붙음 (중급부터)
4. 폴리라인 — LINE만 처리하고 LWPOLYLINE을 무시하면 결선 누락 (초급에서도 가능)
5. 레이어 필터 — 치수선·해칭을 결선으로 오검출해 정밀도 하락 (전 티어)
