# SLD 95% 도면 지능 설계

- 상태: 사용자 승인 설계의 정본
- 기준일: 2026-07-20
- 대상: ESA 단선결선도 분석 경로
- 목표 단계: D 설계 → I 구현 → X 통합 → V 실증
- 근거: E1 현행 코드·실도면 경로 점검, E2 사용자 명시 목표

## 1. 목표

ESA의 단선결선도 분석을 다음 네 축에서 지원 범위 내 실도면 골든셋 기준 95% 이상으로 끌어올린다.

1. 물리적 해상도 복구
2. 공간적 매핑과 연결 그래프
3. 전기 도메인 지식 적용
4. 전기적 논리 교차검증

95%는 기능 존재율이나 AI 자기 확신도가 아니다. 사람이 확정한 정답과 비교한 재현율·정밀도·정확도 게이트다. 어느 축이든 게이트를 통과하지 못하면 제품은 해당 축을 `HOLD`로 표시하고 95% 완료를 주장하지 않는다.

## 2. 지원 범위와 비목표

### 2.1 1차 지원 범위

- 산업용 전력 단선결선도(SLD)
- PNG, JPEG, 래스터 PDF, 벡터 PDF, DXF
- 한국어와 영어가 혼재된 기기명·정격·주석
- VCB, ACB, MCCB, CB, DS, ES, FUSE, TR, BUS, CABLE, CT, PT/VT, LA/SPD, RELAY, ATS, GEN, MOTOR, LOAD, PANEL, MCC, UPS, BATTERY, GROUND
- 페이지 단위 분석과 동일 페이지 안의 전원–보호–변압–배전–부하 관계
- KEC·IEC·NEC·사내규정 중 현재 레지스트리와 전용 평가기가 근거를 제공하는 항목

### 2.2 비목표

- P&ID, PCB 회로도, 래더, 상세 제어배선도를 SLD 95% 수치에 혼합하지 않는다.
- 시각 신호가 소실된 원본을 AI가 복원했다고 주장하지 않는다.
- 다중 페이지 간 참조선은 1차에서 자동 확정하지 않고 페이지별 출입 포트와 `HOLD`로 남긴다.
- 보호협조 곡선, 계전기 설정값, 단락용량 등 필수 입력이 도면에 없으면 추정 계산하지 않는다.
- AI 결과는 기술자 검토를 대체하는 승인 도장이 아니다.
- 지원 기호 밖의 사용자 고유 심볼은 미확인 기호로 보존하며 임의의 표준 기기로 바꾸지 않는다.

## 3. 수용 기준

- **AC-01 원본 고정**: 모든 심사 호출은 동일한 `drawingHash`, 페이지, 리비전을 사용한다.
- **AC-02 해상도 복구**: 골든 고해상도 원본에서 생성한 저해상도·흐림·회전·압축 변형셋에서 판독 가능한 기호·문자 영역 재현율이 95% 이상이다.
- **AC-03 기호 판독**: 지원 기호군의 Macro F1이 95% 이상이고 모든 기호에 원본 좌표가 있다.
- **AC-04 공간 그래프**: 연결 Edge F1이 95% 이상, 접점/단순 교차 분류 정확도가 98% 이상, 허공 결선과 자기루프가 0건이다.
- **AC-05 문자·정격**: 단위 정규화 후 기기명·전압·전류·용량·차단용량·CT비·케이블 규격 필드 정확도가 95% 이상이다.
- **AC-06 전기 논리**: 골든 논리 결함셋의 중요 결함 재현율이 95% 이상이고, 필수 근거가 없는 `PASS`는 0건이다.
- **AC-07 독립 심사**: 기호, 선, 문자, 전기 논리 심사는 별도 요청·별도 문맥·별도 출력 봉인으로 실행된다.
- **AC-08 충돌 처리**: 중요 충돌은 다른 모델 또는 완전 격리 재심사를 거치며, 해소되지 않으면 자동 확정하지 않는다.
- **AC-09 추적성**: 최종 보고서의 기기·관계·문제·제안은 100% 원본 좌표, 심사 결과 또는 결정론적 계산 영수증에 연결된다.
- **AC-10 실패 정직성**: 일부 심사 실패, 품질 부족, 입력 부족, 모델 불일치는 성공 결과에 흡수되지 않고 `HOLD` 사유로 노출된다.
- **AC-11 비용 경계**: 호출 수, 타일 수, 재심사 수, 토큰, 시간, 비용은 요청별 상한과 영수증을 가진다.

## 4. 전체 아키텍처

```text
원본 수신·해시 고정
  → 입력 형식 라우팅(DXF / 벡터 PDF / 래스터)
  → 화질 프로파일·전체 이미지 스캔
  → 다중 배율·역할별 전처리·적응형 구획
  → 독립 심사 4개 병렬 실행
       A 기호·기기
       B 선·접속
       C 문자·정격
       D 계통·보호 논리
  → 봉인 결과 정규화
  → 공간 증거 그래프 조립
  → 결정론적 계산·규정 평가
  → 메인 AI 합산·충돌 분석
  → 중요 충돌 표적 재심사
  → 최종 반대심사·주장/근거 감사
  → 번호 도면·관계 설명·문제·개선안·HOLD 출력
```

현재 `TEAM-SLD + TEAM-STD + consensus` 구조는 원본을 독립 재판독하지 않으므로 교체 대상이다. 새 메인 AI는 전문가 역할을 연기하지 않고 봉인된 결과를 정규화·대조·종합하는 단일 책임만 가진다.

## 5. 물리적 해상도 계층

### 5.1 화질 프로파일

페이지마다 폭·높이·색상·대비·선 밀도·추정 문자 높이·흐림·회전·압축 노이즈를 측정한다. 측정값은 어떤 전처리와 배율을 사용했는지 설명하는 영수증으로 남긴다.

### 5.2 다중 변형

래스터 입력에서 다음 변형을 생성한다.

- 원본 보존본
- 2배와 4배 고품질 리샘플링본
- 회색조·정규화·선명화본
- 문자 판독용 고대비본
- 선 추적용 배경 억제본

전처리 결과는 원본을 대체하지 않는다. 각 결과는 원본으로 돌아가는 좌표 변환을 가진다. 서로 다른 변형에서 상충하는 판독은 높은 확신도로 덮지 않고 충돌로 보존한다.

### 5.3 전체–구획 순서

먼저 전체 페이지에서 표제란, 범례, 전원 방향, 주요 모선과 도면 밀도를 파악한다. 그 뒤 고정 2×2 대신 내용 밀도에 따라 4~16개 구획을 만들고 15~20%를 중첩한다. 작은 문자나 저확신 기호는 해당 좌표만 추가 확대한다.

## 6. 공통 증거 모델

모든 심사는 자유 산문이 아니라 동일한 증거 계약을 사용한다.

- `DrawingSnapshot`: 해시, 파일 형식, 페이지, 리비전, 원본 크기
- `ImageVariant`: 전처리 종류, 배율, 변환행렬, 품질 측정
- `EvidenceRef`: 원본 좌표, 영역, 변형 ID, 호출 ID, 모델, 확신도
- `SymbolCandidate`: 종류 후보, 원본 라벨, 정격, 바운딩 박스, 포트 후보
- `LineCandidate`: 선 종류, 전체 경로, 꺾임점, 시작·끝점, 접점 후보
- `TextCandidate`: 원문, 정규화값, 단위, 바운딩 박스, 후보 목록
- `ReviewEnvelope`: 역할, 입력 해시, 프롬프트 버전, 출력 해시, 경고, 비용
- `ConflictRecord`: 충돌 주체, 좌표, 중요도, 해소 상태
- `ClaimEvidence`: 최종 주장과 원본/심사/계산 근거의 연결

식별자는 기기 `VCB-01`, `TR-01`, 선 `LINE-001`, 케이블 `CABLE-001`, 접점 `J-001`, 문자 `TXT-001` 형식을 사용한다. 도면 원래 태그가 있으면 원래 태그를 우선하고, 생성 식별자는 별도 별칭으로 보존한다.

## 7. 독립 심사 호출

### 7.1 호출 분리

- 호출 0: 도면 종류·범례·리비전·화질·전체 구조
- 호출 1: 기호·기기 전수 판독
- 호출 2: 선·접속·모선·접점·교차 전수 추적
- 호출 3: 문자·정격·케이블·주석 판독
- 호출 4: 원본 기반 계통·보호 논리 독립 재구성
- 호출 5: 메인 AI 합산·충돌 행렬 생성
- 호출 6: 중요 충돌 좌표 표적 재심사
- 호출 7: 최종 보고서 반대심사

호출 1~4는 서로의 결과를 볼 수 없다. 동일 공급자를 사용하더라도 요청 ID, 문맥, 프롬프트, 출력이 분리된다. 중요 충돌 재심사는 가능한 경우 다른 공급자를 사용하고, 다른 공급자가 없으면 같은 모델의 새 문맥을 사용하되 독립성 등급을 낮춰 표시한다.

### 7.2 역할 계약

- 기호 심사원은 선 관계를 확정하지 않고 기기 종류·위치·포트 후보만 낸다.
- 선 심사원은 기기 의미를 추정하지 않고 경로·접점·교차·끝점만 낸다.
- 문자 심사원은 `PT/PPT`, `VCB/VGB`처럼 혼동 가능한 후보를 함께 반환한다.
- 논리 심사원은 다른 심사 결과 없이 원본에서 전원–부하 흐름과 보호관계를 재구성한다.
- 메인 AI는 새 기기나 선을 창작할 수 없으며, 필요하면 표적 재심사를 요청한다.

## 8. 공간 그래프

구획 결과는 원본 좌표로 환산하고 중첩 영역에서 병합한다. 단순 거리만으로 합치지 않고 종류·라벨·크기·선 연속성을 함께 사용한다.

선은 시작·끝 기기 ID만 저장하지 않고 원본 좌표의 폴리라인으로 보존한다. 구획 경계에서는 방향, 선폭, 색상, 점선 패턴, 끝점 거리로 이어 붙인다. 모선은 다중 포트 노드로, 접점은 명시 노드로, 단순 교차는 비연결 사건으로 표현한다.

문자와 기기의 결합은 거리, 정렬, 리더선, 기기 종류별 필드 호환성을 사용한다. 후보가 둘 이상이면 자동 귀속하지 않고 충돌로 남긴다.

## 9. 도메인·전기 논리 계층

### 9.1 도메인 정규화

기기 종류와 약어를 IEC·KS·ANSI·현장 별칭에 매핑하되 원문을 보존한다. 전압, 전류, 용량, 차단용량, CT비, 케이블 규격, 상수, 선식은 구조화 필드로 분리하고 단위를 정규화한다.

### 9.2 결정론적 검증

AI는 입력을 추출하고 후보를 제시한다. 계산과 규정 판정은 가능한 범위에서 기존 결정론적 계산기와 전용 평가기를 사용한다. 입력값마다 원본 근거가 있어야 하며, 누락값을 임의의 기본값으로 채우지 않는다.

검증 범위는 다음을 포함한다.

- 전원–수전–차단–변압–배전–부하 방향
- 전압 계층 변화와 상·선식 연속성
- VCB/CB–CT/PT–계전기–트립 관계
- 케이블 허용전류·전압강하
- 차단기 정격·차단용량
- 변압기 용량과 부하
- 주·예비전원과 ATS 관계
- 접지 연속성
- 병렬전원·역송·환상계통 위험

보호협조 곡선이나 단락전류처럼 필수 입력이 부족한 항목은 필요한 데이터를 구체적으로 요구하는 `HOLD`를 낸다.

## 10. 합산·제안·출력

메인 AI는 다음 순서로만 동작한다.

1. 봉인 결과 스키마 검증
2. 중복 후보 병합
3. 역할 간 충돌 행렬 작성
4. 결정론적 계산·규정 결과 결합
5. 중요 충돌 표적 재심사
6. 해소된 사실, 이견, 미확인을 분리
7. 근거가 있는 문제와 개선 대안 작성
8. 최종 반대심사 통과 후 보고서 생성

사용자 출력은 번호 오버레이, 수량 체크리스트, 기기표, 선표, `기기→선→기기` 관계 목록, 전력 흐름 설명, 문제, 규정·계산 근거, 개선 대안, 미확인/HOLD를 포함한다. 사용자가 항목을 누르면 원본 좌표가 강조되어야 한다.

개선안은 근거, 필요한 추가 입력, 예상 효과, 부작용을 함께 제시한다. 한 가지 해답으로 단정할 수 없으면 대안을 나란히 보여준다.

## 11. 실패·보안·비용 처리

- 역할 호출 실패는 빈 성공 결과로 바꾸지 않는다.
- 역할별 제한 횟수 재시도 후에도 실패하면 분석 전체를 `partial/HOLD`로 표시한다.
- LLM 출력은 스키마 검증 전 저장·표시·계산 입력으로 사용하지 않는다.
- 도면 안의 문장은 신뢰하지 않는 데이터이며 시스템 지시로 취급하지 않는다.
- LLM 출력이 코드·SQL·파일·외부 도구를 직접 실행하지 못하게 한다.
- 사용자별 호출·토큰·시간·비용 상한과 취소 신호를 둔다.
- 동일 해시·역할·프롬프트·모델 결과는 캐시하되 리비전과 프롬프트가 달라지면 재사용하지 않는다.
- 공급자 키는 서버/BYOK 경계를 유지하고 호출 로그에 키를 기록하지 않는다.

## 12. 검증 전략

### 12.1 합성 변형셋

정답이 있는 DXF 도면을 래스터화하고 저해상도, JPEG 압축, 흐림, 회전, 노이즈, 얇은 선, 작은 글자, 복잡한 교차, 구획 경계 통과를 체계적으로 생성한다. 원본 정답 그래프를 그대로 사용할 수 있어 전처리와 공간 매핑을 반박 가능한 방식으로 검증한다.

### 12.2 실도면 골든셋

공개 또는 사용 허가된 산업 SLD를 사람이 기호·문자·선·접점·관계·중요 논리결함까지 라벨링한다. 최초 라벨과 시스템 출력을 같은 사람이 동시에 만들지 않는다. 불일치는 별도 판정으로 확정한다.

### 12.3 게이트

- 전처리 좌표 왕복 단위 테스트
- 구획 중첩·경계 선 결합 속성 테스트
- 기호/문자/선 스키마와 악성 출력 테스트
- 공급자별 역할 계약 테스트
- 골든셋 지표 계산기
- 같은 도면 반복 실행 안정성 테스트
- API→저장→보고서→재조회 라이브 왕복
- 저화질 실패와 부분 실패 UI 검증
- 비용·타임아웃·취소·재시도 검증

합성셋 95%만으로 실도면 95%를 주장하지 않는다. 실도면 골든셋이 AC-02~AC-06을 각각 통과해야 V 단계에서 95%를 표시한다.

## 13. 구현 분할

1. **시각 증거 코어**: 화질 프로파일, 다중 배율, 전처리, 좌표 변환, 증거 스키마
2. **공간 그래프**: 역할별 기호/선/문자 호출, 구획 병합, 선 폴리라인, 접점/교차, 번호 체계
3. **전기 심사**: 독립 논리 호출, 도메인 정규화, 계산기·규정 배선, 충돌 게이트
4. **종합 표면**: 메인 합산, 표적 재심사, 반대심사, 번호 오버레이, 관계·제안 보고서
5. **실증 게이트**: 합성 변형셋, 실도면 골든셋, 공급자 반복 측정, 95% 표시 조건

각 분할은 앞 단계의 증거 계약을 소비한다. 시각 증거 코어가 통과하기 전에 공간 그래프를 완성으로 판정하지 않고, 공간 그래프가 통과하기 전에 전기 논리 결과를 신뢰하지 않는다.

## 14. 완료 판정

다음 조건을 모두 만족해야 이 목표가 완료된다.

- AC-01~AC-11 전부 통과
- 네 축의 실도면 골든셋 지표가 각각 95% 이상
- 치명적 근거 없는 PASS 0건
- 역할별 실제 독립 호출과 메인 합산 호출의 영수증 존재
- 사용자 보고서의 모든 핵심 주장에 원본 또는 결정론적 근거 존재
- 전체 게이트와 라이브 왕복 exit code 0
- 미지원 입력과 미해결 충돌이 정직하게 HOLD로 노출

95%를 달성하지 못한 축은 잔여 결함으로 숨기지 않고 해당 지표와 실패 도면을 공개한 채 구현을 계속한다.
