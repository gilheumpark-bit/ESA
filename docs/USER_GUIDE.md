# ESVA 사용자 가이드

> 기준일: 2026-07-23 · 대상: ESVA v0.2.0 내부·공개 교보재 검증 버전

ESVA는 “답을 대신 확정하는 AI”가 아니라, 전기 엔지니어가 계산식·입력·판본·경고를 다시 확인할 수 있게 만드는 작업대입니다. 숫자가 나왔다는 이유만으로 설계 승인이나 법규 적합 판정으로 사용하지 마십시오.

## 1. 가장 빠른 시작

1. 상단의 **계산기**에서 목적에 맞는 계산기를 고릅니다.
2. 입력 단위와 계통 조건을 확인하고 계산합니다.
3. 결과 아래의 공식, 계산 단계, 경고, 적용 규격 판본을 확인합니다.
4. 저장이 필요하면 로그인한 뒤 영수증 또는 프로젝트에 보관합니다.
5. 안전·법규 관련 결과는 현행 공인 원문과 책임 엔지니어 검토로 마감합니다.

계정이나 AI 키가 없어도 결정론적 계산기와 로컬 기준서 탐색은 사용할 수 있습니다. 로그인·영구 저장·AI 검색은 배포 환경 설정에 따라 제한될 수 있습니다.

## 2. 화면별 역할

| 화면 | 할 수 있는 일 | 조건 |
|---|---|---|
| 홈 | 검색, 계산기·도면·기준서 진입, 일반 질문의 AI 답변 열기 | AI 답변은 공급자 키 필요 |
| 검색 | 로컬 전기공학 자료와 계산기 탐색, 조건부 AI 응답과 계산 영수증 | AI 응답은 공급자 키 필요 |
| 계산기 | 등록된 57개 계산기 실행 | 없음 |
| 기준서 | 저장소의 KEC·NEC·IEC·JIS·NER·ESA 스냅샷 탐색 | 현행 원문 자동 동기화 아님 |
| SLD 도구 | 이미지·DXF·PDF 분석, 전체 문서 판독, 정정·재개·전문팀 검토 | 형식별 제한, Vision 분석은 AI 키 필요 |
| Studio | 파일 기반 전문팀 검토 또는 무파일 AI 질문 | 입력 데이터와 AI/서버 환경에 따라 제한 |
| 프로젝트 | 계산·자료 묶음 저장, 공유 토큰 또는 초대 | 로그인·Supabase 필요 |
| 영수증·보고서 | 입력/출력/근거/경고/해시 확인 | 저장된 항목은 소유권 또는 공개 설정 필요 |
| 커뮤니티 | 질문·답변·투표 | 로그인 및 DB 필요 |
| 현장 모드 | 체크리스트 완료·SOS 기록 | 로그인 및 DB 필요. 긴급 서비스 대체 아님 |
| 설정 | 언어·국가·BYOK·온프레미스 연결 | 일부 기능은 배포 관리자 설정 필요 |

관리자 화면은 서버가 확인한 Enterprise 사용자만 접근할 수 있습니다. SSO·결제·알림 발송은 화면의 존재와 실제 배포 연결 상태를 구분해야 합니다.

## 3. 계산기 사용법

계산기마다 입력 계약과 유효 범위가 다릅니다. 다음 순서로 확인하십시오.

- 전압, 전류, 길이, 상수, 역률과 단위를 먼저 확인합니다.
- 선택한 국가·규격과 실제 관할 규정이 같은지 확인합니다.
- 경고 또는 HOLD가 있으면 누락 입력과 근거 데이터를 보완합니다.
- 계산 영수증의 입력·공식·중간 단계·엔진 버전·규격 판본을 보관합니다.
- 아크플래시, 단락전류, 보호협조 등 안전 관련 결과는 ETAP·SKM 등 승인된 도구 또는 별도 수계산으로 교차 검증합니다.

ESVA는 각 계산기를 공식별 기준값과 입력 계약으로 테스트합니다. “모든 계산기가 동일한 ±오차를 보장한다”는 의미는 아닙니다. 측정 오차, 제조사 데이터, 온도·포설 조건, 반올림 방식은 코드 테스트와 별개의 현장 불확실성입니다.

## 4. 기준서와 판정

기준서 화면은 저장소에 포함된 판본 스냅샷을 보여 줍니다. 2026년 현재 KEC·NEC·IEC의 최신 개정이 자동 반영되는 서비스가 아닙니다.

- 조항 검색 결과의 제목·번호·판본을 공인 원문과 대조합니다.
- 임계값이나 전용 평가기가 없으면 ESVA는 PASS/FAIL을 추측하지 않고 HOLD를 반환합니다.
- NER·ESA를 포함한 일부 조항은 검색은 가능하지만 입력 데이터에 따라 판정이 제한될 수 있습니다.
- 영수증의 “현행 규격 여부”는 공인 판본과 확인일이 검증되지 않으면 `아니오`로 표시됩니다.

설계·시공·감리 전에는 관할 기관이 공표한 현행 원문을 확인하십시오.

## 5. AI 검색과 BYOK

AI 답변은 계산 엔진이나 법적 판정을 대체하지 않습니다. 응답에 숫자·규격·안전 단정이 포함되면 출처와 계산 근거를 확인하십시오. 출력 필터가 근거가 부족한 표현을 차단하거나 사람 검토 대상으로 돌릴 수 있습니다.

### AI 계산 질문

전압강하처럼 지원되는 계산 질문에 필수 입력을 모두 적으면 ESVA가 AI 호출 전에 정본 계산기를 실행합니다. 화면의 `ESA 계산기 실행` 표시는 계산기 ID, 입력과 결과가 실제 응답에 포함됐다는 뜻입니다. AI는 이 영수증을 설명하지만 영수증 밖의 새 숫자를 만들어서는 안 됩니다.

필수 입력이 빠지면 수치 계산을 실행하지 않습니다. 이때는 누락 입력과 다음 계산 순서만 확인하십시오. 계산 영수증이 없는데 AI 문장에 확정 수치가 있다면 결과를 사용하지 말고 오류로 신고하십시오.

BYOK 키 저장 방식은 다음과 같습니다.

1. 브라우저가 원문 키를 Web Crypto AES-GCM으로 암호화합니다.
2. 추출 불가능한 암호화 `CryptoKey`를 IndexedDB에 저장합니다.
3. 암호문만 `localStorage`에 보관합니다.
4. AI 요청 시 브라우저가 키를 복호화해 TLS로 ESVA 서버에 보내고, 서버가 공급자 호출에 일시 사용합니다.
5. ESVA 애플리케이션은 원문 키를 데이터베이스나 서버 로그에 저장하지 않습니다.

설정 화면의 **연결 확인**도 브라우저에서 공급자 주소를 임의 호출하지 않습니다. ESVA의 same-origin 중계 API가 공급자별 고정 공식 엔드포인트에 키를 일회성으로 전달하고, 응답 본문이나 키를 저장하지 않은 채 유효 여부만 돌려줍니다.

따라서 “키가 서버를 전혀 통과하지 않는다”는 구조는 아닙니다. 신뢰할 수 없는 배포 주소나 공용 PC에서는 BYOK를 사용하지 말고, 사용 후 설정에서 키를 삭제하십시오. 브라우저 저장소가 초기화되면 키도 다시 등록해야 합니다.

온프레미스 공급자는 관리자가 `ONPREMISE_ALLOWED_ORIGINS`에 정확한 origin을 허용한 경우에만 서버가 연결합니다. 임의 URL, 사용자정보가 포함된 URL, 허용되지 않은 사설 주소는 차단됩니다.

## 6. 도면 분석

### 이미지

PNG/JPEG 이미지는 지원 AI 공급자가 기기와 연결을 추출합니다. 낮은 해상도, 겹친 심볼, 비표준 표기, 손글씨는 오인식될 수 있습니다.

### DXF

DXF는 벡터 엔티티와 CAD 블록을 파싱합니다. 좌표를 직접 읽는다는 것은 픽셀 추정이 없다는 뜻이지, 모든 블록을 100% 인식한다는 뜻은 아닙니다. 커스텀 블록명, 외부 참조, 깨진 엔티티는 누락될 수 있습니다. 파일 한도는 50MB입니다.

### PDF

단일 벡터 PDF 분석은 선택 페이지의 선·문자를 파싱합니다. 스캔 이미지로만 된 PDF에는 벡터 정보가 없으므로 이 단순 경로는 실패합니다. SLD 전체 문서 판독에서 Vision 공급자를 선택하면 페이지를 이미지로 렌더해 역할별 분석을 시도합니다. 저해상도 OCR과 연결 판독은 PARTIAL 또는 HOLD가 될 수 있습니다. 파일 한도는 경로별 API 계약을 확인하십시오.

### 전체 문서와 구획 번호

SLD 전체 문서 판독은 페이지를 빠짐없이 열거한 뒤 전체 이미지와 정밀 구획을 함께 봅니다. 결과 번호는 기기명 자체가 아니라 근거를 추적하기 위한 ID입니다.

- `Pxx-Axx`: 논리 구획
- `Pxx-Cxxx`: 구획 경계에서 잘린 연결선
- `Pxx-Uxxx`: 짝을 확정하지 못한 연결 끝점
- `Pxx-S/L/T/R`: 기호, 선, 문자, 역할 근거

경계선이 있다고 해서 별도 부품으로 읽지 않습니다. 최종 보고서는 전체 도면을 다시 보며 구획별 결과를 합치고 중복과 오병합을 제거합니다. `U`와 HOLD가 남으면 원본에서 해당 번호의 반대쪽 연결과 표기를 확인하십시오.

### 전문팀 검토

ESVA는 계통도팀, 평면도팀, 기준서팀의 세 전문 분석 결과를 모은 뒤 별도 합의 단계에서 충돌을 정리합니다. 합의 단계는 독립적인 네 번째 전문가가 아닙니다. 서로 다른 전문팀 두 곳 이상이 정상 완료되지 않으면 합의 완료로 표시하지 않고 사람 검토 필요 상태를 남깁니다.

도면 결과의 `confidence`, 고립 노드, 연결 문제, 가정, HOLD 항목을 모두 확인하십시오. 자동 생성된 계산 순서는 실제 명판값과 부하 조건이 없으면 설계 확정값이 아닙니다.

비로그인 팀 검토 보고서는 현재 브라우저 세션에서만 다시 열릴 수 있습니다. 서버 저장과 새 세션 재조회가 필요한 경우 로그인·Supabase가 구성된 배포인지 확인하십시오.

## 7. 영수증과 보고서

계산 영수증에는 가능한 범위에서 다음을 기록합니다.

- 원본 입력과 단위
- 출력과 계산 단계
- 사용 공식과 규격 참조
- 경고·권고·면책 문구
- 엔진·판본 메타데이터
- SHA-256 무결성 해시

조회 시 저장된 내용으로 해시를 다시 계산해 변조 여부를 검사합니다. SHA-256 해시는 파일의 동일성을 확인하는 장치이며, 제3자 공증이나 법적 서명을 자동으로 의미하지 않습니다.

비공개 영수증은 소유권을 확인한 사용자만 볼 수 있습니다. `is_public=true`로 명시된 영수증만 로그인 없이 조회할 수 있습니다.

## 8. 저장·공유·삭제

- 비로그인 계산은 브라우저 임시 저장 상태에 남을 수 있습니다.
- 로그인 사용자의 프로젝트·영수증·커뮤니티·현장 기록은 구성된 Supabase에 저장됩니다.
- 프로젝트 공개 공유는 추측하기 어려운 공유 토큰으로 조회하며, 비공개 프로젝트 ID만으로는 열 수 없습니다.
- 계정 삭제는 서버가 확인한 사용자 ID 범위의 데이터만 대상으로 하며, 운영 백업 보존은 배포 정책을 따릅니다.
- 분석 이벤트는 DNT(`Do Not Track`) 또는 `esva-analytics-opt-out=true`일 때 전송하지 않습니다. 활성 상태에서는 이벤트 종류·동작·시간·세션 ID와 서버 로그의 IP가 처리될 수 있습니다.

정확한 처리 범위는 앱의 개인정보 처리방침 화면을 함께 확인하십시오.

## 9. 조건부·비활성 기능

다음 항목은 화면이나 코드가 존재해도 별도 인프라 검증 전에는 완전한 운영 기능으로 보지 않습니다.

- Stripe 구독: 테스트 모드 키, 가격 ID, 서명 검증 웹훅, DB 반영 왕복 필요
- Weaviate 검색: 호스트·인증·스키마 프로비저닝 필요
- 이메일·푸시 알림: 발송자 인프라 미연결
- IPFS 타임스탬프 등록: 기본 비활성. 블록체인 거래·제3자 공증·법적 서명이 아님
- 다중 인스턴스 레이트 리밋: 공유 저장소 또는 신뢰 프록시 필요

비활성 기능이 성공한 것처럼 보이거나 임시 데이터로 대체되면 버그로 신고해 주십시오.

## 10. 자주 묻는 질문

### 결과가 HOLD인 이유는 무엇인가요?

필수 입력, 공인 임계값, 적용 판본 또는 서로 다른 전문팀 결과가 부족하다는 뜻입니다. HOLD는 실패 점수가 아니라 추측을 거부한 상태입니다.

### AI 키를 등록하면 사용량이 무제한인가요?

아닙니다. ESVA 자체의 요청 제한과 공급자 계정의 요금·쿼터·정책이 모두 적용됩니다. BYOK는 공급자 비용과 한도를 없애지 않습니다.

### 계산기는 오프라인에서도 되나요?

계산 로직 자체는 외부 AI를 요구하지 않지만, 현재 앱이 완전한 오프라인 PWA로 검증됐다는 뜻은 아닙니다. 최초 페이지 로드, 로그인, 저장, 검색 인프라에는 네트워크가 필요할 수 있습니다.

### DXF/PDF 분석 결과를 바로 시공에 써도 되나요?

안 됩니다. 인식 누락과 연결 오류를 사람이 확인하고, 명판값·보호기기 설정·관할 규격을 원본 도면과 대조해야 합니다.

### 버그나 기준서 오류는 어디에 신고하나요?

앱의 **문의** 화면을 사용하십시오. 재현 경로, 입력값, 기대 결과, 실제 결과, 규격 판본을 함께 적으면 확인이 빠릅니다. 민감정보와 실제 API 키는 첨부하지 마십시오.
