1. 프로젝트 개요 (ESA 브랜딩)
==================================================

브랜드명: ESA (Electronic Search AI)
  E - Electronic (전기/전자 엔지니어링)
  S - Search (검색, 탐색)
  A - AI (인공지능)

슬로건:
  "The Engineer's Search Engine"
  "Know the Standard. Trust the Answer."
  "전기 엔지니어의 AI 파트너"

도메인 후보: esa.ai / esa-engine.com / esa.io

한 줄 정의:
ESA는 전기 엔지니어링 특화 글로벌 AI 검색 플랫폼이다.
전 세계 전기 규격 (KEC/IEC/NEC/JIS/GB/AS-NZS/EN 등) 완전 지원 +
글로벌 논문/규격/뉴스 실시간 수집 + 할루시네이션 없는 출처 기반 답변 +
영수증 계산기 + 날짜 박제 + 협업 + 현장 모바일 + 법적 공증 +
해외 플랜트 규격 자동 변환까지 원스톱 엔지니어링 플랫폼이다.

시장 기회:
  가트너 예측: 2026년까지 전통 검색엔진 볼륨 25% 감소
  버티컬 LLM 시장: 2025년 2.9B달러 -> 2033년 18.7B달러 (CAGR 26%)
  전기 특화 버티컬 검색: 전 세계 공백 상태
  해외 플랜트 EPC 시장: 한국 3대 EPC사 연간 수십조 규모

목표 사용자:
  1차 (국내): 전기 엔지니어, 수험생, 전력 R&D 연구자
  2차 (국내+글로벌): EPC사, 전력공기업, 엔지니어링 컨설팅사
  3차 (글로벌): 일본/중국/미국/유럽/호주 전기 엔지니어

2. 핵심 철학 및 원칙
==================================================

원칙 1: 할루시네이션 제로
  AI = 수집된 문서 기반 요약만
  출처 없는 답변 = 시스템 레벨 차단
  수치/규격 = 계산 엔진 직접 연산

원칙 2: 날짜 박제
  모든 데이터에 타임스탬프 필수
  언제 기준인지 모르면 현장에서 못 쓴다

원칙 3: 영수증 투명성
  계산 과정 전체 공개
  출처 규격 조항까지 명시

원칙 4: 법적 명확성
  모든 산출물에 면책 조항 자동 삽입
  공증은 시점 증명이지 결과 보증이 아님

원칙 5: 글로벌 규격 동등성
  어떤 국가 규격으로 질문해도 답변 가능
  국가 간 규격 자동 변환 및 차이점 명시

원칙 6: 저작권 준수 (v6.0 추가)
  규격 원문 무단 크롤링 금지
  공개 API / 공식 라이선스 / 요약 참조만 허용
  원문은 출처 링크 제공, ESA DB에는 색인만

원칙 7: 도메인 플러그인
  전기 -> 기계 -> 토목 -> 화학 순차 확장

원칙 8: Hybrid BYOK
  Free: 플랫폼 기본 소형 LLM
  Pro 이상: 사용자 본인 API 키 선택

원칙 9: 법적 방패
  날짜 박제 + 블록체인 공증 (익명화)
  = 당시 현행 규격 기준 타임스탬프 증명

5. 인증 시스템 - NEW v6.0
==================================================

인증 방식:

소셜 로그인 (일반 사용자):
  Google OAuth 2.0
  GitHub OAuth (개발자/연구자 타겟)
  Apple Sign In (모바일 iOS)
  Microsoft OAuth (기업 사용자)

이메일/패스워드 (대안):
  이메일 인증 필수
  비밀번호 bcrypt 해싱
  이메일 인증 링크 24시간 유효

Enterprise SSO (v6.0):
  SAML 2.0 연동 (대기업/공기업)
  LDAP / Active Directory 연동
  온프레미스 인증 서버 지원
  예: KEPCO, 삼성엔지니어링, 현대엔지니어링

JWT 토큰 전략:
  Access Token: 만료 15분 (단기)
  Refresh Token: 만료 30일 (장기, HttpOnly Cookie)
  토큰 갱신: Refresh Token으로 자동 갱신
  토큰 블랙리스트: Redis 저장 (로그아웃 처리)
  토큰 구조: {user_id, tier, domain_pref, country_code, iat, exp}

다국가 가입 정책 (v6.0):
  한국: 개인정보보호법 준수 (만 14세 이상)
  EU: GDPR 준수 (명시적 동의, 쿠키 배너)
  일본: APPI 준수 (개인정보 취급 동의)
  중국: PIPL 준수 (데이터 현지화 요건 검토)
  미국: COPPA (13세 미만 불가)
  호주: Privacy Act 준수

세션 관리:
  동시 세션 제한:
    Free: 2개
    Pro: 5개
    Team: 10개
    Enterprise: 무제한
  의심 세션 감지 (다른 국가 IP 동시 접속 시 알림)

인증 DB 스키마:

테이블 auth_providers:
  id: UUID PK

  provider: VARCHAR(20) - google, github, apple, microsoft, email, saml

테이블 refresh_tokens:
  id: UUID PK
  user_id: UUID FK users
  token_hash: VARCHAR(64)
  expires_at: TIMESTAMP
  is_revoked: BOOLEAN
  device_info: JSONB
  created_at: TIMESTAMP

테이블 sessions:
  id: UUID PK
  user_id: UUID FK users
  session_token: VARCHAR(200)
  ip_address: INET
  country_code: VARCHAR(10)
  user_agent: TEXT
  created_at: TIMESTAMP
  last_active_at: TIMESTAMP
  expires_at: TIMESTAMP

6. 플랜 가드 로직 - NEW v6.0
==================================================

개념:
  Free/Pro/Team/Enterprise 각 플랜별 제한을
  API 레벨에서 실시간으로 enforce하는 시스템

카운터 저장소: Redis

Redis 키 구조:
  usage:calc:{user_id}:{YYYY-MM} -> 계산기 사용 횟수 (월 단위)
  usage:search:{user_id}:{YYYY-MM} -> 검색 횟수
  usage:cert:{user_id}:{YYYY-MM} -> 블록체인 공증 횟수
  usage:convert:{user_id}:{YYYY-MM} -> 규격 변환 횟수
  usage:export:{user_id}:{YYYY-MM} -> Export 횟수
  plan:{user_id} -> 현재 플랜 정보 (캐시, 5분 TTL)

플랜별 제한 테이블:

항목                   Free    Pro     Team    Enterprise
계산기 월 사용 횟수     20회    무제한   무제한   무제한
검색 결과 개수          3건     전체     전체     전체
뉴스 지연              1주     실시간   실시간   실시간
블록체인 공증           불가    10회/월  무제한   무제한
협업 인원               1인     5인      무제한   무제한
국가 규격 비교          2개국   전체     전체     전체
규격 변환               불가    10회/월  무제한   무제한
Excel Export           불가    가능     가능     가능
API 접근               불가    불가     가능     가능
동시 세션               2개     5개      10개     무제한

가드 적용 방식:

1단계: API Gateway에서 JWT 디코딩 -> 플랜 확인
2단계: Redis에서 현재 사용량 조회
3단계: 제한 초과 시 429 응답 + 업그레이드 안내
4단계: 허용 시 사용량 카운터 증가 (INCR)
5단계: 월 초 자동 리셋 (Celery Beat 스케줄러)

플랜 업그레이드/다운그레이드 처리:

업그레이드 즉시:
  새 플랜 적용
  Redis 플랜 캐시 갱신
  남은 Free 제한 해제

다운그레이드 (결제 주기 만료 후):
  현재 주기 끝까지 Pro 유지
  만료 7일 전 알림
  다운그레이드 시 진행 중인 프로젝트:
    읽기 전용으로 전환 (삭제 안 함)
    30일 내 Export 또는 업그레이드 권장
    30일 후 아카이브 (복구 가능)

테이블 plan_limits:
  tier: VARCHAR(20) PK
  calc_monthly: INT (-1 = 무제한)
  search_count: INT
  news_delay_days: INT
  cert_monthly: INT
  collab_members: INT
  country_compare_count: INT
  convert_monthly: INT
  excel_export: BOOLEAN
  api_access: BOOLEAN
  max_sessions: INT

테이블 usage_logs:
  id: UUID PK
  user_id: UUID FK users
  action: VARCHAR(50)
  year_month: VARCHAR(7) - 2026-04
  count: INT
  updated_at: TIMESTAMP

  Paddle (글로벌 세금 처리 대행)

    EU 디지털서비스세 대응
    일본 소비세 / 호주 GST 처리

Enterprise:
  인보이스(후불) 방식 지원
  연간 계약 선불 할인
  구매발주서(PO) 기반 결제

구독 관리:

신규 구독:
  결제 정보 등록 -> 즉시 플랜 활성화
  첫 결제 실패 시 3일 유예 후 Free 전환

갱신 실패 처리:
  갱신 실패 D+0: 이메일 알림
  갱신 실패 D+3: 재시도 + 알림
  갱신 실패 D+7: 재시도 + 강조 알림
  갱신 실패 D+14: Free 전환 + 데이터 보존 30일

환불 정책:
  월 구독: 3일 이내 전액 환불
  연간 구독: 30일 이내 전액, 이후 일할 계산
  Enterprise: 계약서 기준

DB 스키마:

테이블 subscriptions:
  id: UUID PK
  user_id: UUID FK users
  tier: VARCHAR(20)
  billing_cycle: VARCHAR(10) - monthly, yearly
  status: VARCHAR(20) - active, past_due, cancelled, trialing
  current_period_start: TIMESTAMP
  current_period_end: TIMESTAMP
  stripe_subscription_id: VARCHAR(200)
  toss_subscription_key: VARCHAR(200)
  created_at: TIMESTAMP

테이블 invoices:
  id: UUID PK

  허용전류 (온도 기준 보정)

  차단기 정격 (IEC <-> UL-ANSI)
  접지 저항 기준
  변압기 임피던스 기준

변환 신뢰도 점수 (v6.0):
  0.95~1.0: 직접 변환 가능 (단위만 다름)
  0.80~0.95: 가정/보정 필요 (온도 기준 등)
  0.60~0.80: 유사 개념 매핑 (주의 필요)
  0.60 미만: 변환 불가 -> 재설계 권장 표시

변환 불가 알림:
  전압 체계 자체가 다를 때 (22.9kV -> 6.6kV)
  규격 간 해당 개념 없을 때
  -> 변환 불가 이유 + 대안 제시

EPC 특화:
  발주처별 규격 요구사항 등록
  사우디 아람코: NEC + SAES 기준
  UAE ADNOC: IEC + ADCO 기준
  변환 결과 영문 PDF 보고서 생성

34. 에러 처리 및 Fallback 전략 - NEW v6.0
==================================================

핵심 원칙:
  어떤 컴포넌트가 실패해도 사용자에게 명확한 안내
  가능한 경우 자동 Fallback으로 서비스 유지
  모든 에러는 로그 저장 + 알림

Weaviate 벡터 검색 실패:
  1차: 자동 재시도 (3회, 지수 백오프)
  2차: PostgreSQL 키워드 검색으로 Fallback
  3차: 검색 불가 안내 + 수동 규격 링크 제공
  알림: 관리자 Slack 알림 즉시

Vision-LLM API 타임아웃 (30초 초과):
  1차: 자동 재시도 (2회)
  2차: 원본 텍스트 청킹만으로 저장
  3차: "표/수식 파싱 지연 중 - 텍스트만 제공" 표시
  백그라운드 큐에 재처리 작업 등록

BYOK 키 만료/오류:
  즉시 감지: API 응답 401/429 감지
  사용자 알림: "API 키가 만료되었거나 잔액이 부족합니다"
  자동 Fallback: 플랫폼 기본 LLM으로 전환 (Pro는 중급 LLM)
  키 재입력 안내: 설정 페이지 바로가기 버튼 제공
  유예 기간: 24시간 기본 LLM 임시 제공

IPFS 공증 저장 실패:
  1차: 자동 재시도 (3회)
  2차: 대체 IPFS 노드 시도 (다중 노드 구성)
  3차: 임시 보류 상태 저장
       "공증 처리 중입니다. 완료 시 알림 드립니다."
  복구: 배치 작업으로 24시간 내 재시도

계산 엔진 오류:
  수식 실행 오류: 즉시 감지 + 구체적 오류 메시지
  "입력값이 유효 범위를 벗어났습니다 (전압 범위: 100V-765kV)"
  수치 오버플로우: 안전 처리 후 경고 표시
  영수증 생성 불가 시: 중간 계산값이라도 표시

PDF/Excel Export 실패:
  1차: 재시도
  2차: 단순 포맷으로 Fallback (스타일 없는 기본 포맷)

  도메인 특화

  전문가 검수 매핑 DB 기반

  표 파싱: 88% / 수식 파싱: 90~95%

뉴스 번역/요약: 88~93%
자격증/일정: 95% 이상
OCR 명판 인식: 90% 이상 (다국어 포함)

향상 전략:
  단기: A/B 테스트 / 수치 범위 검증 / 규격 매핑 전문가 검수
  중기: 피드백 루프 / 커뮤니티 버그 리포트
  장기: 전기 특화 LLM 파인튜닝 / 지식 그래프 고도화

   언제 어떤 규격/국가 기준인지 타임스탬프 영구 증명

   KEC / IEC / NEC / CEC / EN / DIN VDE / BS 7671
   NF C / JIS / 내선규정 / GB / AS-NZS + 저작권 완전 준수

4. 해외 플랜트 규격 자동 변환
   KEC -> NEC / IEC / JIS 원클릭 변환 + 신뢰도 점수

5. 국가별 전압/전기 체계 DB
   계산기 국가 선택 -> 표준전압/주파수/규격 자동 적용

6. 단위 변환 엔진
   mm2 <-> AWG <-> kcmil 자동 변환

7. 규격 저작권 완전 준수 (v6.0)
   오픈/요약/링크만 분류 -> 법적 분쟁 없는 운영

8. 인증 + 플랜 가드 완성 (v6.0)
   SSO/SAML Enterprise / JWT / 티어별 실시간 제한

9. 에러 처리 완전 설계 (v6.0)
   모든 컴포넌트 Fallback -> 서비스 중단 없음

10. 다국 개인정보법 대응 (v6.0)
    GDPR / APPI / PIPL / 개인정보보호법 완전 대응

11. 결제 시스템 완성 (v6.0)

    규격서 표/수식까지 완전 색인
