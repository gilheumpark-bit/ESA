1. 프로젝트 개요
==================================================

브랜드: ESA (Electronic Search AI)
  E - Electronic (전기/전자 엔지니어링)
  S - Search (검색)
  A - AI (인공지능)

슬로건: The Engineer's Search Engine

운영 방식:
  eh-universe 플랫폼과 완전 독립 운영
  별도 도메인 (esa.ai 또는 esa-engine.com)
  별도 Vercel 배포
  내부적으로만 코드 재활용 (인증/결제/배포)
  겉으로는 완전 별개 서비스

한 줄 정의:
  전기 엔지니어링 특화 글로벌 AI 플랫폼
  전 세계 전기 규격 완전 지원 +
  할루시네이션 없는 출처 기반 답변 +
  영수증 계산기 + 날짜 박제 + 협업 +
  현장 모바일 + 법적 공증 + 글로벌 규격 변환

시장 기회:
  버티컬 LLM 시장: 2025년 2.9B달러 -> 2033년 18.7B달러
  전기 특화 AI 플랫폼: 전 세계 공백
  eh-universe 기반: 4개국어 이미 지원

목표 사용자:
  1차: 전기 엔지니어, 수험생, R&D 연구자
  2차: EPC사, 전력공기업, 엔지니어링 컨설팅사
  3차: 일본/중국/미국/유럽/호주 전기 엔지니어

2. 핵심 철학 및 원칙
==================================================

원칙 1: 할루시네이션 제로
  AI = 수집된 문서 기반 요약만
  출처 없는 답변 = 시스템 레벨 차단
  수치/규격 = 계산 엔진 직접 연산

원칙 2: 날짜 박제
  모든 데이터에 타임스탬프 필수
  언제 기준인지 모르면 현장에서 못 쓴다

원칙 3: 영수증 투명성
  계산 과정 전체 공개
  출처 규격 조항까지 명시
  AI가 계산했다 X -> 엔진이 직접 돌린 결과

원칙 4: 법적 명확성
  모든 산출물에 면책 조항 자동 삽입
  공증은 시점 증명이지 결과 보증이 아님

원칙 5: 글로벌 규격 동등성
  어떤 국가 규격으로 질문해도 답변 가능
  단위 체계 자동 변환

원칙 6: 저작권 준수
  규격 원문 무단 크롤링 금지
  공개 API / 공식 라이선스 / 요약 참조만

원칙 7: 특허 안전 구조
  수학 공식 = 공공 도메인 (특허 불가)
  구현 코드 = 독자 저작물
  영수증 시스템 = ESA 독자 특허 출원 검토

원칙 8: 심플 우선
  제목 + 링크 방식 (뉴스/논문/제품)
  저작권 리스크 제로
  구현 난이도 최소

원칙 9: Hybrid BYOK
  Free: 플랫폼 기본 소형 LLM
  Pro 이상: 사용자 본인 API 키

5. AI 자동 라우팅 + 샌드박스 분리
==================================================

핵심 원칙:
  검색창 하나
  탭 없음
  AI가 질문 보고 알아서 라우팅

샌드박스 구조:

Weaviate Collection 분리:
  collection: general_search
    일반 웹 문서 / 전체 뉴스
  collection: electrical
    KEC / IEC / 전기 논문 / 전기 뉴스
  collection: ai_ml
    AI 논문 / 도구 비교
  collection: mechanical (Phase 2)
  collection: fire_safety (Phase 2)

라우팅 로직:

사용자 질문
    ↓
의도 분류기 (경량 모델 - 빠른 처리)
    ↓
전기 관련 키워드 감지:
  전압 / 전류 / 변압기 / 케이블
  KEC / IEC / 차단기 / 접지
  수변전 / 태양광 / ESS / 역률
    ↓
Yes -> 전기 샌드박스
No  -> 일반 샌드박스
애매 -> 전기 우선 + 일반 보조

라우팅 예시:
  "변압기 용량 계산해줘" -> 전기 샌드박스 + 계산기 추천
  "오늘 날씨" -> 일반 샌드박스
  "GaN 인버터 최신 논문" -> 전기 샌드박스
  "SiC 반도체" -> 전기 + AI 샌드박스 동시
  "YouTube URL" -> 유튜브 요약 엔진

DB 스키마:

테이블 search_sandboxes:
  id: UUID PK
  name: VARCHAR(50) - general / electrical / ai_ml
  collection: VARCHAR(100) (Weaviate 컬렉션명)
  keywords: TEXT[] (라우팅 키워드)
  is_active: BOOLEAN
  created_at: TIMESTAMP

테이블 routing_logs:
  id: UUID PK
  user_id: UUID
  query: TEXT
  detected_sandbox: VARCHAR(50)
  confidence: FLOAT
  created_at: TIMESTAMP

7. 계산기 완전 설계 (특허 안전 구조)
==================================================

특허 안전 원칙:

수학 공식 = 공공 도메인 (특허 불가)
  e = √3 × I × L × R / 1000
  → 100년 된 공식
  → 누구나 구현 가능

구현 코드 = ESA 독자 저작물
  다른 SW 코드 복사 금지
  처음부터 독자 구현

UI = 독자 설계
  경쟁사 레이아웃 참고만
  완전히 다르게 설계

영수증 시스템 = ESA 독자 특허 출원 검토
  계산 과정 단계별 공개 +
  규격 조항 자동 매핑 +
  날짜 박제 통합 구조
  → 세계 최초 조합
  → 특허 출원 가능

코드 레벨 안전장치:

각 계산기 파일 상단 명시:
  수학적 근거: 공식 명시
  규격 근거: KEC/IEC 조항 명시
  공식 자체는 공공 도메인
  본 구현 코드는 ESA 독자 저작물

계산기 엔진 레이어 구조:

사용자 입력 (자연어 또는 폼)
    ↓
파라미터 추출 레이어
  자연어 -> 구조화 데이터
  단위 변환 (Pint 라이브러리)
  입력값 유효성 검증
    ↓
계산 엔진 레이어
  NumPy / SciPy / SymPy
  공식 직접 구현 (특허 없음)
  단계별 중간값 로깅
    ↓
검증 레이어
  결과값 범위 검증
  규격 기준 적합 여부
  경고/권장사항 생성
    ↓
영수증 생성 레이어
  단계별 계산 과정 공개
  날짜 박제
  규격 조항 명시
  면책 조항 삽입
    ↓
출력 레이어
  화면 표시
  PDF / Excel Export
  블록체인 공증
  공유 링크 생성
