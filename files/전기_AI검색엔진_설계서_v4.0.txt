==================================================

전기 엔지니어링 특화 AI 플랫폼.
글로벌 논문/규격/뉴스 실시간 수집 + 할루시네이션 없는 출처 기반 답변
+ 영수증 계산기 + 날짜 박제 + 협업 + 현장 모바일 + 법적 공증까지
원스톱 엔지니어링 플랫폼.

2. 핵심 철학 및 원칙
==================================================

원칙 1: 할루시네이션 제로
  AI = 수집된 문서 기반 요약만 수행
  출처 없는 답변 = 시스템 레벨에서 차단
  수치/규격 = 계산 엔진이 직접 연산

원칙 2: 날짜 박제
  모든 데이터에 타임스탬프 필수
  원문 발행일 / 수집일 / 검증일 / 규격 버전 / 계산 실행 시각 / 엔진 버전
  "언제 기준인지 모르면 현장에서 못 쓴다"

원칙 3: 영수증 투명성
  계산 과정 전체 공개
  출처 논문/규격 조항까지 명시
  AI가 계산했다 X -> 엔진이 직접 돌린 결과

원칙 4: 법적 명확성 (v4.0 추가)
  모든 산출물에 면책 조항 자동 삽입
  "본 계산서는 참고용이며 최종 책임은 검토자에게 있습니다"
  공증은 시점 증명이지 결과 보증이 아님을 명시

원칙 5: 도메인 플러그인
  전기 -> 기계 -> 토목 -> 화학 -> 전체 공학
  코어 건드리지 않고 도메인만 추가

원칙 6: Hybrid BYOK (v4.0 업데이트)
  Free: 플랫폼 기본 소형 LLM 제공 (Llama/Mistral 계열)
  Pro 이상: 사용자 본인 API 키 (GPT-4o, Claude, Gemini)
  API 키는 서버에 암호화 저장, 로그에 절대 기록 안 함

원칙 7: 법적 방패
  날짜 박제 + 블록체인 공증 (익명화 적용)
  = 당시 현행 규격 기준으로 작성됨을 타임스탬프로 영구 증명
  블록체인 공증 전 민감 정보 철저 익명화 (GDPR 대응)

7. PDF 표/수식 파싱 전략 (Vision-LLM) - NEW v4.0
==================================================

문제점:
  IEC 규격서, IEEE 논문의 핵심 정보는 복잡한 수식과 다단 표에 존재
  기본 텍스트 청킹으로는 이를 온전히 벡터 DB에 담을 수 없음
  단순 OCR도 수식 의미 손실 발생

해결 전략: Vision-LLM 전처리 파이프라인

처리 단계:

1단계: PDF 페이지 이미지 변환
  PyMuPDF 또는 pdf2image 로 페이지별 이미지 추출
  해상도: 300 DPI (수식 인식 최적)

2단계: 콘텐츠 타입 분류
  텍스트 페이지: 기존 파서로 처리
  표/수식 포함 페이지: Vision-LLM 전달

3단계: Vision-LLM 처리 (GPT-4o Vision 또는 Claude 3.5 Sonnet)
  프롬프트:
    "이 이미지에서 표와 수식을 추출하세요.
     표는 Markdown 형식으로 변환하세요.
     수식은 LaTeX 형식으로 변환하세요.
     전기공학 도메인 전문 용어를 정확히 유지하세요."

4단계: 구조화 저장
  표 -> Markdown 변환 후 parsed_tables JSONB 저장
  수식 -> LaTeX 변환 후 parsed_formulas JSONB 저장
  변환 결과 -> 텍스트와 함께 벡터 임베딩

5단계: 검색 연동
  수식 검색: "전압강하 공식" 검색 시 LaTeX 수식 포함 결과 반환
  표 검색: "케이블 허용 전류 표" 검색 시 표 데이터 반환
  계산기 연동: 추출된 수식 -> 계산기 자동 생성 가능성 검토

비용 최적화:
  모든 페이지에 Vision-LLM 적용하면 비용 폭발
  표/수식 포함 페이지만 선별 적용
    선별 기준: 페이지 이미지에 수식/표 감지 휴리스틱 (면적 비율)
  처리 결과 캐싱: 동일 DOI 문서 재처리 방지

처리 결과 품질 관리:
  LaTeX 수식 유효성 검증
  표 행/열 개수 일관성 확인
  변환 실패 시 원본 이미지 링크로 대체 제공

22. 블록체인 공증 (익명화 강화) - UPDATED v4.0
==================================================

필요성:
  전기 설계 계산서 법적 보존 기간: 준공 후 10년 이상
  위변조 불가 타임스탬프 증명 필요

GDPR 및 개인정보 충돌 해결 (v4.0 핵심):

문제:
  IPFS에 저장된 데이터는 삭제 불가 (잊힐 권리와 충돌)
  영수증에 기업 기밀/개인정보 포함 가능

해결책:
  공증 대상: 익명화된 계산 수치와 메타데이터만
  공증 제외: 프로젝트명, 담당자 이름, 기업명, 기밀 수치

익명화 처리 단계:
  1단계: 민감 정보 식별
    프로젝트명 -> [PROJECT_ID_HASH]
    담당자명 -> [USER_ID_HASH]
    기업명 -> [ORG_ID_HASH]
    특정 지역/시설명 -> [LOCATION_HASH]

  2단계: 익명화 데이터셋 생성
    입력 수치: 유지 (계산 재현에 필요)
    계산 과정: 유지 (수식/단계 검증에 필요)
    결과값: 유지
    규격 참조: 유지
    민감 식별 정보: 해시로 대체

  3단계: 익명화 데이터 SHA-256 해시 생성
  4단계: IPFS 저장 (익명화 데이터셋)
  5단계: 블록체인 타임스탬프 (해시 + 시각)

  원본 영수증: 플랫폼 DB에만 보관 (사용자 삭제 요청 시 삭제 가능)
  IPFS: 익명화 버전만 (삭제 불가하나 개인정보 없음)

공증 프로세스:
  영수증 완성
  -> 사용자 면책 조항 동의 확인
  -> 익명화 처리
  -> SHA-256 해시 생성 (익명화 데이터 기준)
  -> IPFS 분산 저장
  -> 블록체인 타임스탬프
  -> 공증 완료 배지 부착
  -> 검증 URL 제공

검증 결과 표시:
  영수증 ID: receipt_abc123
  익명화 해시: a3f2c1...8d4e
  블록: #12345678
  시각: 2026.04.04 14:32:11 KST
  규격: KEC 2024년판
  결과: 검증 완료 - 익명화 원본과 일치, 위변조 없음
  주의: 본 공증은 계산 시점과 수치의 무결성을 증명하며 결과의 정확성을 보증하지 않습니다

25. Hybrid BYOK 구조 - UPDATED v4.0
==================================================

배경:

  해결: Hybrid 구조로 진입 장벽 완화

Hybrid BYOK 구조:

Free / 기본 사용자:
  플랫폼 기본 LLM 제공 (Llama 3.x 또는 Mistral 자체 호스팅)
  기능: 검색/요약 기본 기능
  품질: 중간 수준
  비용: 플랫폼 부담 (소형 모델이므로 저렴)
  목적: 서비스 가치 먼저 체감하게 함

Pro 이상 사용자:
  본인 BYOK 키 입력 (선택)
  GPT-4o / Claude Sonnet / Gemini Pro 등 고품질 모델
  기능: 전체 기능 + 고품질 답변
  비용: 사용자 부담

BYOK 미입력 Pro 사용자:
  플랫폼 제공 중급 LLM (GPT-4o mini 또는 Claude Haiku)
  플랫폼이 API 비용 일부 부담 (Pro 구독료로 충당)

기업/Enterprise:
  전용 API 키 또는 온프레미스 LLM 연동
  Azure OpenAI / AWS Bedrock 등 기업 계약 모델 연동 가능

LLM 품질 티어:

  Free: Llama 3.x (자체 호스팅) - 기본 검색/요약
  Pro: Claude Haiku / GPT-4o mini - 향상된 답변
  Pro + BYOK: GPT-4o / Claude Sonnet - 최고 품질
  Enterprise: 커스텀 모델 연동

BYOK 키 관리:
  사용자 API 키 입력
  -> 클라이언트 AES-256 암호화
  -> 암호화 키만 서버 전송

27. 글로벌 규격 DB (NEC 추가) - UPDATED v4.0
==================================================

지원 규격 (v4.0):

한국:
  KEC (한국전기설비규정) - 전기설비 핵심
  전기사업법 / 기술기준 / 판단기준
  KS C 시리즈

국제:
  IEC 60364 시리즈 (저압 설비)
  IEC 61850 (디지털 변전소)
  IEC 60076 (변압기)
  IEC 62619 (배터리)
  IEEE Std 시리즈

미국 (v4.0 추가):
  NEC - National Electrical Code (NFPA 70)
  ANSI/NEMA 시리즈
  IEEE C 시리즈 (보호계전)
  UL 표준

독일/유럽:
  DIN VDE 0100 시리즈
  EN 규격

일본:
  JIS C 시리즈
  내선규정

KEC vs IEC vs NEC 비교 DB:

개념: 전압강하 허용 기준
  KEC 232.52: 간선 3%, 분기 2%
  IEC 60364-5-52: 권장 4% 이내
  NEC Article 210: 일반 5%, 권장 3%
  차이점: NEC가 가장 완화된 기준, KEC가 가장 엄격

개념: 케이블 허용 전류
  KEC 232.3: 온도 40도C 기준
  IEC 60364-5-52: 온도 30도C 기준
  NEC Table 310: 온도 40도C 기준 (AWG 체계)
  차이점: 단위 체계 다름 (mm2 vs AWG)

개념: 접지 저항 기준
  KEC 140: 계통별 기준 상이
  IEC 60364-4-44: TN/TT/IT 계통별
  NEC Article 250: 25 ohm 이하 원칙
  차이점: 계통 분류 방식 다름

EPC 기업 특화 기능:
  해외 플랜트 수출 시 NEC 기준 계산서 생성
  KEC와 NEC 동시 준수 여부 자동 체크
  규격 충돌 시 경고 및 더 엄격한 기준 적용 권장

31. 정확도 목표
==================================================

영역별 목표 정확도:

계산기 (영수증): 99.5% 이상
  근거: Python 엔진 직접 연산, AI 계산 아님

AI 검색 (RAG): 85~92%
  근거: 도메인 특화 RAG, Perplexity 대비 5~10%p 높음

Vision-LLM PDF 파싱 (v4.0): 88~95%
  근거: 표 구조 인식 88%, 수식 인식 90~95%
  오차 케이스: 복잡한 다단 표, 손으로 쓴 수식

뉴스 번역/요약: 88~93%
  근거: 원문 출처 병기로 검증 가능

자격증/일정: 95% 이상
  근거: 큐넷 공식 데이터 크롤링

OCR 명판 인식: 90% 이상
  근거: 정형 데이터 특성

정확도 향상 전략:

단기:
  청크 크기 A/B 테스트
  전기 도메인 임베딩 파인튜닝
  수치 범위 자동 검증
  Vision-LLM 파싱 결과 품질 검증 자동화

중기:
  사용자 피드백 루프 (틀린 답변 신고)
  커뮤니티 버그 리포트 반영
  전문가 검수 레이어 추가

장기:
  전기 특화 LLM 파인튜닝
  지식 그래프 기반 정확도 향상
  Vision-LLM 단선결선도 인식 고도화
