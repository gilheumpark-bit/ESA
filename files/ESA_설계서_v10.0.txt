1. 프로젝트 개요
==================================================

브랜드: ESA (Electronic Search AI)
슬로건: The Engineer's Search Engine

운영:
  eh-universe와 완전 독립
  별도 도메인 / 별도 Vercel 배포
  내부 코드만 재활용
    인증 / 결제 / 배포 / 4개국어 / 에이전트 프레임워크

한 줄 정의:
  전기 엔지니어링 특화 글로벌 AI 플랫폼
  전 세계 전기 규격 완전 지원 +
  할루시네이션 없는 출처 기반 답변 +
  영수증 계산기 + 날짜 박제 + BYOK 기본

시장:
  버티컬 LLM: 2025년 2.9B -> 2033년 18.7B (CAGR 26%)
  전기 특화 AI: 전 세계 공백
  eh-universe: 4개국어 이미 구축

목표 사용자:
  1차: 전기 엔지니어 / 수험생 / R&D
  2차: EPC사 / 전력공기업 / 컨설팅사
  3차: 일본/중국/미국/유럽/호주 엔지니어

2. 핵심 철학 및 원칙
==================================================

원칙 1: 할루시네이션 제로
  AI = 수집된 문서 기반 요약만
  수치/규격 = 계산 엔진 직접 연산

원칙 2: 날짜 박제
  모든 데이터 타임스탬프 필수
  언제 기준인지 모르면 현장에서 못 쓴다

원칙 3: 영수증 투명성
  계산 과정 전체 공개
  출처 규격 조항까지 명시

원칙 4: 법적 명확성
  모든 산출물 면책 조항 자동 삽입
  공증 = 시점 증명, 결과 보증 아님

원칙 5: 글로벌 규격 동등성
  어떤 국가 규격으로 질문해도 답변 가능

원칙 6: 저작권 준수
  제목 + 링크만 (뉴스/논문/제품)
  저작권 리스크 제로

원칙 7: 특허 안전
  수학 공식 = 공공 도메인
  구현 코드 = ESA 독자 저작물
  영수증 시스템 = 특허 출원 검토

원칙 8: BYOK 기본
  지금은 사용자 본인 API 키
  플랫폼 LLM 비용 제로
  계산기는 BYOK 없이도 작동

원칙 9: 샌드박스 격리
  국가 × 장르 × 기능 완전 격리
  교차 오염 구조적 방지

원칙 10: 심플 우선
  메인 화면 = 구글처럼
  검색창 하나
  AI가 알아서 처리

5. 에이전트 아키텍처
==================================================

3계층 구조:

계층 1: 메인 에이전트 (오케스트레이터)
  항상 켜져 있음
  stateless 설계 (요청별 독립 컨텍스트)
  단일 창구 (사용자 직접 대면)
  역할:
    질문 분석
    라우팅 결정
    브릿지 호출 여부 판단
    최종 답변 통합

계층 2: 브릿지 에이전트 (조율자)
  교차 질문 시만 가동
  끝나면 즉시 종료
  역할:
    필요 샌드박스 목록 파악
    병렬 async 호출
    결과 수집 + 중복 제거
    통합 후 메인에 반환
  가동 조건:
    전기 + AI 교차
    여러 국가 동시 비교
    에이전트 워크플로우
    여러 장르 동시 필요

계층 3: 샌드박스 에이전트 (전문가)
  호출 시만 가동
  독립 격리 운영
  직접 통신 금지 (반드시 상위 경유)
  역할:
    해당 도메인 검색
    계산기 실행
    규격 조회
    결과 반환

메인 에이전트 stateless 설계:
  요청마다 독립 컨텍스트
  세션 ID 기반 격리
  처리 후 컨텍스트 즉시 소멸
  SPOF 없음 (여러 인스턴스 가능)
  Vercel Functions로 자동 해결

브릿지 타임아웃:
  각 샌드박스 최대 3초
  초과 시 해당 결과 제외
  나머지로 답변 생성
  "일부 결과 제외됨" 표시

격리 차원 3개:
  차원 1: 국가 (한국/일본/미국/중국/호주/중동)
  차원 2: 장르 (전기/AI/규격/자격증/뉴스)
  차원 3: 기능 (검색/계산/뉴스/커뮤니티)

격리 레벨:
  Level 1: DB 격리 (Weaviate 컬렉션 분리)
  Level 2: 프롬프트 격리 (시스템 프롬프트 독립)
  Level 3: 툴 격리 (에이전트별 툴셋 분리)
  Level 4: 메모리 격리 (대화 이력 독립)
  Level 5: 네트워크 격리 (Enterprise - 컨테이너)

샌드박스 통신 규칙:
  메인 -> 하위: 질문 + 컨텍스트만 전달
  하위 -> 메인: 결과만 반환
  샌드박스 간 직접 통신: 완전 금지
  교차 질문: 메인/브릿지가 각각 질의 후 통합

중복 데이터 관점:
  같은 IEC 논문이 여러 샌드박스에 있어도
  컨텍스트 / 언어 / 관점이 다름
  = 중복이 아닌 의도적 다각도 색인
  진짜 중복 = 동일 샌드박스 내 중복
  → DOI 기준 중복 제거로 해결

콜드 스타트 예열:
  자주 쓰는 샌드박스: 상시 웜업
    한국 전기 / 일반 검색
  가끔 쓰는 샌드박스: 요청 전 프리워밍
  거의 안 쓰는 샌드박스: 콜드 (5~10초 허용)

모니터링:
  샌드박스별 상태 대시보드
  🟢 정상 / 🟡 느림 / 🔴 다운
  자동 재시작 (3회 실패 시 관리자 알림)

7. AI 자동 라우팅
==================================================

원칙:
  검색창 하나 / 탭 없음
  AI가 질문 보고 자동 라우팅

라우팅 로직:
  질문 입력
      ↓
  의도 분류기 (경량 모델)
      ↓
  전기 키워드 감지:
    전압/전류/변압기/케이블/KEC/IEC
    차단기/접지/수변전/태양광/ESS/역률
      ↓
  단순 질문 -> 해당 샌드박스 직접
  교차 질문 -> 브릿지 가동
  YouTube URL -> 유튜브 요약 엔진
  일반 질문 -> 일반 샌드박스

오판 보완:
  결과 화면에 샌드박스 표시
  "전기 도메인으로 검색됨"
  사용자가 직접 변경 가능
  재검색 버튼 제공

검색어 자동완성:
  전기 전문 용어 사전 기반
  KEC 조항 번호 직접 검색
  동의어 처리
    MCCB = 배선용 차단기
    VCB = 진공 차단기
    TR = 변압기

8. 대장르 / 소장르 체계
==================================================

대장르 1: 전기 (Electrical)
  전력 시스템 (발전/송전/배전/수변전)
  전기 설비 (옥내배선/동력/조명/접지)
  보호 협조 (차단기/계전기/단락/지락)
  신재생 에너지 (태양광/풍력/ESS/수소)
  전력 전자 (인버터/컨버터/SiC/GaN)
  스마트 그리드 (AMI/HVDC/디지털변전소)
  EV/모빌리티 (충전/V2G/전기추진)
  전기 안전 (감전/피뢰/방폭)

대장르 2: AI/ML
  모델 (LLM/비전/음성/멀티모달)
  학습 (지도/강화/파인튜닝)
  인프라 (GPU/데이터센터/엣지)
  AI 도구 비교
  전기 x AI 교차

대장르 3: 규격/법령
  한국 / 국제 / 미국 / 일본 / 중국 / 유럽 / 호주

대장르 4: 자격증
  한국 / 일본 / 미국 / 유럽 / 호주

대장르 5: 제품/기술
  변압기 / 차단기 / 케이블 / 배전반
  인버터 / 태양광 / 배터리 / 계측

대장르 6: 뉴스/트렌드
  전력 산업 / 신재생 / 반도체 / EV
  정책/규제 / AI 기술 / 글로벌 규격 개정

대장르 7: 계산기
  전력 기초 / 전압강하 / 케이블
  변압기 / 보호 협조 / 접지
  전동기 / 신재생 / 수변전 / 조명
  글로벌 전용 / AI 특화

9. 계산기 완전 설계 (특허 안전)
==================================================

특허 안전 원칙:
  수학 공식 = 공공 도메인 (특허 불가)
  구현 코드 = ESA 독자 저작물 (독자 구현)
  UI = 독자 설계 (경쟁사 참고만)
  영수증 시스템 = 특허 출원 검토 (세계 최초 조합)

계산기 엔진 레이어:
  입력 레이어 (자연어 또는 폼)
      ↓
  파라미터 추출 (단위 변환 + 유효성 검증)
      ↓
  계산 엔진 (NumPy/SciPy/SymPy - 공식 직접 구현)
      ↓
  검증 레이어 (범위 검증 + 규격 적합 + 경고)
      ↓
  영수증 생성 (단계별 공개 + 날짜 박제 + 면책)
      ↓
  출력 (화면 + PDF/Excel + 공증 + 공유)

입력 방식 분기:
  [자연어 입력] - BYOK 필요
    "변압기 용량 22.9kV 100A 계산해줘"
  [폼 직접 입력] - BYOK 불필요
    전압: [22900] V / 전류: [100] A
  → 계산기는 BYOK 없이도 핵심 기능 사용 가능

엣지 케이스 처리:
  규격에 모든 보정 계수 명시되어 있음
  KEC 232.3: 온도보정 -> 집합보정 순서 명시
  유효성 검사로 비현실 입력 차단
  규격 충돌 시 엄격한 기준 적용 + 경고
